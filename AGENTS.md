# AGENTS.md

本文件定义本仓库内 Codex/AI 代理的默认协作规则，作用域为仓库根目录及其全部子目录。

## 1. 开发流程

- 默认采用 spec-driven 方式推进：先对齐任务，再落地代码，再做最小验证。
- 每次只聚焦当前任务，避免无关改动。
- 未经用户明确指令，不主动提交 commit。

## 2. 学习笔记机制（强约束）

- 必须维护 `learning-notes/` 学习笔记体系。
- 每完成一个模块任务（如 A2、A3、B1），自动新增或更新对应模块笔记。
- 文件按阶段分类：
  - `learning-notes/phase-a/`
  - `learning-notes/phase-b/`
  - 后续阶段按需扩展。
- 文件命名建议：`<任务编号>-<主题>.md`，例如：`A2-pytest-and-smoke-test.md`。
- 每完成一个模块，模块笔记需固定包含：模块功能、在整体流程中的位置、模块作用。

## 3. 用户提问沉淀（强约束）

- 用户在开发过程中的关键问题，必须整理进对应模块笔记的「常见问题」或同等章节。
- 回答后应同步更新笔记，不依赖用户二次提醒。

## 4. 用户偏好记录

- 持续维护 `learning-notes/USER_PREFERENCES.md`。
- 当用户新增协作习惯或输出偏好时，及时追加更新。

## 5. 输出风格

- 说明尽量清晰、简洁、可执行。
- 涉及命令、路径、文件名时，给出可直接复制的形式。
- 完成任务后，说明改动文件与验证结果。

## 6. 代码注释语言（强约束）

- 仓库内所有代码注释必须使用中文。
- 包括但不限于：行内注释、块注释、函数/类文档字符串（docstring）中的注释性描述。
- 新增或修改代码时，若存在英文注释，需同步改为中文。
- 英语技术术语（如 MCP、LLM、Embedding、Vector Store、Dashboard）保留英文，不强制翻译。

## 7. 协作执行偏好

- 常用开发命令默认自动执行，不需要逐条确认（如 `git add`、`git commit`、`git status`、`powershell Get-Content`）。
- 定向测试默认自动执行，不需要单独确认。
