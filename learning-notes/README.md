# 学习笔记目录说明

## 目标

用于持续沉淀本项目学习过程：

- 每完成一个模块（如 A2、A3、B1）就新增/更新对应笔记文件。
- 你在对话里提的关键问题，也会整理进对应模块笔记中的「常见问题」部分。

## 目录约定

- `learning-notes/phase-a/`：阶段 A 笔记
- `learning-notes/phase-b/`：阶段 B 笔记
- 后续按阶段继续扩展（phase-c, phase-d...）

## 文件命名约定

- 格式：`<任务编号>-<主题>.md`
- 例如：`A2-pytest-and-smoke-test.md`

## 维护规则（从现在开始）

1. 每完成一个 DEV_SPEC 任务，自动补充该任务对应笔记。
2. 若你在该任务阶段提出问题，会同步整理到该任务笔记中。
3. 尽量保持每个任务一份独立笔记，避免信息混杂。
